# 🧠 블로그 글 생성 규칙 최종본

## 1️⃣ 기본 원칙

1. 모든 글은 처음 보는 사람도 이해할 수 있을 만큼 쉽게 설명한다.
2. 그러나 어투는 어린아이 말투가 아니라, 부드럽고 신뢰감 있는 전문가의 어조를 유지한다.
3. 설명은 항상 다음 3가지를 포함한다.
   - 왜 필요한지
   - 어떻게 하는지
   - 언제 쓰는지
4. 문제 해결형 실무 콘텐츠를 우선한다.
5. 추상적인 설명보다 실제 작업 흐름을 중심으로 작성한다.

---

## 2️⃣ 글 구조 고정

모든 글은 반드시 아래 구조를 따른다.

1. 서론
2. 본문 1: 개념 설명
3. 본문 2: 실무 적용 + 단계별 설명
4. Q&A
5. 결론

---

## 3️⃣ 분량 규칙

- 서론, 본문 1, 본문 2, Q&A, 결론의 순수 텍스트 합산은 최소 5,000자 이상이어야 한다.
- 목록, 표, 코드, 이미지 설명은 글자 수 계산에서 제외한다.
- 각 섹션은 충분히 자세하게 작성한다.
- 설명 중간 단계를 생략하지 않는다.

---

## 4️⃣ 예제 규칙

- 예제는 반드시 3개만 작성한다.
- 예제는 실무 상황 기반으로 작성한다.
- 각 예제는 아래 요소를 포함한다.
  - 상황 설명
  - 왜 사용하는지
  - 단계별 방법
  - 주의사항

---

## 5️⃣ Q&A 규칙

- Q&A는 반드시 5개만 작성한다.
- 질문은 서로 중복되면 안 된다.
- 각 답변은 최소 5~8문장 이상 작성한다.
- 각 답변은 아래 흐름을 따른다.
  1. 핵심 요약
  2. 해결 방법
  3. 예시
  4. 주의사항
  5. 실전 팁

---

## 6️⃣ 설명 방식 규칙

모든 설명은 반드시 다음 기준을 따른다.

- 단계별로 끊어서 설명한다.
- 메뉴 위치를 포함한다.
- 클릭 순서를 포함한다.
- 어떤 창이 열리는지 설명한다.
- 어떤 옵션을 선택하는지 설명한다.
- 중간 단계를 생략하지 않는다.

예시:

데이터 탭 → 정렬 버튼 → 사용자 지정 정렬 창 → 정렬 기준 선택 → 확인

---

## 7️⃣ 한글/영문 병렬 규칙

모든 글은 반드시 한글/영문 상하 병렬 구조로 작성한다.

작성 순서:

한국어 문단

English paragraph

한국어 문단

English paragraph

규칙:

- 문단 단위로 번역한다.
- 문장 단위 번역은 하지 않는다.
- 모든 섹션에 동일하게 적용한다.
- 서론, 본문 1, 본문 2, Q&A, 결론 모두 한글/영문 병렬로 작성한다.

---

## 8️⃣ 절대 금지 사항

- 설명 생략 금지
- 추상적인 설명 금지
- 단순 정보 나열 금지
- "대충", "적당히" 같은 표현 금지
- 참고 형식 그대로 복사 금지
- 참고 형식의 예제 개수 그대로 따라 하기 금지
- 참고 형식의 Q&A 개수 그대로 따라 하기 금지
- 단일 언어로만 작성 금지

---

🚨🚨🚨 절대 규칙 🚨🚨🚨

아래 참고 형식은 구조 참고용일 뿐이며, 다음 규칙을 반드시 따른다.

1. 예제는 반드시 3개만 작성한다.
2. Q&A는 반드시 5개만 작성한다.
3. 모든 글은 한글/영문 병렬로 작성한다.
4. 참고 형식과 충돌할 경우 항상 위 규칙을 따른다.
5. 참고 형식의 문장, 예제, Q&A를 그대로 재사용하지 않는다.

이 규칙을 어기면 잘못된 결과로 간주한다.

---

## 🚨 9️⃣ 참고 형식

❗ 아래 내용은 절대 그대로 따라 작성하지 말 것.

아래 예시는 “형태 참고용”이다.  
예제 개수, Q&A 개수, 문장 스타일, 단일 언어 구조는 따라 하지 않는다.

---

### 참고해야 할 것

- 서론에서 문제 상황을 먼저 제시하는 흐름
- 본문 1에서 개념을 설명하는 흐름
- 본문 2에서 실제 사용 방법을 단계별로 설명하는 흐름
- 예제에서 상황 → 이유 → 방법 → 주의사항 순서로 설명하는 흐름
- Q&A에서 사용자가 실제로 궁금해할 질문을 다루는 흐름
- 결론에서 핵심을 짧게 정리하는 흐름

---

### 참고하면 안 되는 것

- 참고 예시의 예제 개수
- 참고 예시의 Q&A 개수
- 참고 예시의 문장 그대로 복사
- 참고 예시의 주제 그대로 재사용
- 참고 예시의 단일 한국어 구조

## 10️⃣ 짧은 참고 예시

주제: 엑셀 색상 기준 정렬 방법

### 서론 예시

엑셀에서 색상 기준으로 데이터를 정렬해야 하는 상황은 자주 발생합니다.  
예를 들어 긴급 업무는 빨간색, 확인이 필요한 업무는 노란색, 완료된 업무는 초록색으로 표시해 두었다면 색상 순서대로 정리하고 싶어질 수 있습니다.  
이때 일반 오름차순 정렬을 사용하면 원하는 결과가 나오지 않습니다.  
색상은 값이 아니라 셀의 서식 정보이기 때문에 사용자 지정 정렬 메뉴에서 따로 설정해야 합니다.

In Excel, there are many situations where you need to sort data by color.  
For example, urgent tasks may be marked in red, tasks that need review in yellow, and completed tasks in green.  
In that case, you may want to organize the rows by color order.  
A normal ascending sort will not work properly because color is formatting information, not a cell value.

---

### 본문 1 예시

색상 정렬은 셀 안의 숫자나 글자가 아니라 셀의 배경색 또는 글자색을 기준으로 순서를 바꾸는 기능입니다.  
따라서 숫자 정렬, 가나다 정렬과는 작동 방식이 다릅니다.  
핵심은 정렬 창에서 “정렬 기준”을 값이 아니라 셀 색으로 바꾸는 것입니다.

Color sorting changes the order of rows based on the cell background color or font color, not the number or text inside the cell.  
That means it works differently from number sorting or alphabetical sorting.  
The key is to change the sort basis from value to cell color in the custom sort window.

---

### 본문 2 예시

예제 1. 긴급 업무를 맨 위로 올리기

상황: 업무 목록에서 빨간색으로 표시된 긴급 업무를 가장 위로 올리고 싶습니다.  
방법: 표 전체 선택 → 데이터 탭 → 정렬 버튼 → 정렬 기준을 해당 열로 선택 → 정렬 기준을 셀 색으로 변경 → 빨간색 선택 → 위에 표시 → 확인 순서로 진행합니다.  
주의사항: 특정 열만 선택하면 행 전체가 함께 움직이지 않아 데이터가 어긋날 수 있습니다.

Example 1. Move urgent tasks to the top

Situation: You want to move urgent tasks marked in red to the top of your task list.  
Method: Select the entire table → Data tab → Sort button → choose the target column → change the sort basis to cell color → choose red → place it on top → click OK.  
Caution: If you select only one column, the entire row may not move together, and your data can become mismatched.

---

### Q&A 예시

Q1. 색상 정렬은 일반 정렬과 무엇이 다른가요?

색상 정렬은 셀 안의 값이 아니라 색상 정보를 기준으로 순서를 바꾸는 기능입니다.  
일반 정렬은 숫자나 글자를 기준으로 작동합니다.  
따라서 색상으로 표시한 업무 상태를 정리하려면 사용자 지정 정렬을 사용해야 합니다.  
특히 업무 우선순위, 확인 상태, 완료 여부를 색으로 구분한 경우에 유용합니다.  
정렬 전에는 반드시 표 전체가 선택되어 있는지 확인해야 합니다.

Q1. How is color sorting different from normal sorting?

Color sorting changes the order based on color information, not the value inside the cell.  
Normal sorting works with numbers or text.  
So if you want to organize task status marked by color, you need to use custom sorting.  
It is especially useful when priority, review status, or completion status is marked with colors.  
Before sorting, you should always check that the entire table is selected.

---

### 결론 예시

색상 기준 정렬의 핵심은 셀 값을 보는 것이 아니라 셀의 서식을 기준으로 정렬한다는 점입니다.  
사용자 지정 정렬 창에서 셀 색을 선택하면 업무 우선순위나 상태를 훨씬 쉽게 정리할 수 있습니다.  
다만 표 전체 선택, 병합 셀 제거, 정렬 기준 확인은 반드시 함께 확인해야 합니다.

The key point of color sorting is that it sorts by cell formatting, not by cell value.  
By choosing cell color in the custom sort window, you can organize task priority or status much more easily.  
However, you should always check the full table selection, remove merged cells, and confirm the sort criteria.
